MoreSore Fitness — Project Overview

MoreSore Fitness is a full-stack web application where users can:

  • Log workouts with detailed exercise data (sets, reps, weight, rest, notes).
  • Log meals and daily macros (calories, protein, carbs, fats).
  • Use an AI agent to automatically generate structured workouts and meal entries from natural language.
  • Track progress over time for specific exercises using custom analytics (volume, 1RM estimates, average weight).
  • Upload “pump pics” to visually track physical progress.

Under the hood, it’s driven by Django models and serializers that enforce structure on AI outputs, a DigitalOcean droplet running Gunicorn + Nginx, and an n8n workflow that orchestrates LLM calls and callback APIs back into my application.

System Architecture

The platform is designed as a set of cooperating services:

High-Level Flow

  • User interacts with a Django-rendered dashboard and chatbot.
  • Chatbot sends a request to an n8n AI agent.
  • n8n calls an LLM API (e.g., Gemini) and parses the response into clean JSON.
  • n8n POSTs the structured data back to Django via a callback URL.
  • Django serializers validate, normalize, and save the data to Postgres/SQLite.
  • Updated workouts and meals appear on the user’s dashboard and progress views.

Tech Stack

  • Backend: Django, Django REST Framework, Python.
  • Automation: n8n workflow for LLM orchestration and callbacks.
  • Frontend: Django templates + Tailwind CSS.
  • Database: Django ORM (SQLite / Postgres-ready models).
  • Deployment: DigitalOcean droplet, Gunicorn, Nginx, systemd services.
  • AI: LLM API integration (via n8n) with JSON-based contracts.

Architecture Diagram

AI Agent & n8n Workflow

The platform uses two, almost identical n8n-powered AI agent that handles both workout and meal generation. Django sends a structured request to n8n, which calls the LLM, validates the response, and sends clean JSON back into the app via a callback endpoint.

AI Agent Responsibilities

  • Accepts natural language requests for workouts and meals from the chatbot.
  • Prompts the LLM to return strictly valid JSON with all required fields.
  • Normalizes units and types (numbers, dates, defaults) in a code node.
  • POSTs the structured data back to a Django callback URL.
  • Supports both workout templates and detailed meal/macros logging.

Integration with Django

  • Django serializers validate every field before saving to the database.
  • Workout data is split into workouts, exercises, and per-set details.
  • Meal data is stored as MealEntry objects linked to a DailyLog.
  • Failures or malformed responses are rejected instead of silently saved.

Backend & Data Modeling

  • Custom Django models for Exercise, Workout, WorkoutExercise, MealEntry, DailyLog, and ExerciseProgress.
  • Serializer classes that validate AI outputs and prevent malformed data from being saved.
  • Logic to compute volume, total sets, total reps, average weight, and estimated 1RM per exercise per day.
  • Daily logs aggregate meals and workouts with macro totals.

Frontend

  • Responsive dashboard built with Django templates and Tailwind CSS.
  • Chatbot section where users can talk to the AI to generate workouts or meals.
  • Workout preview editor that lets users adjust sets/reps/weights before saving.
  • Progress dashboard that groups performance by exercise across multiple sessions.
  • “Pictures” upload grid for visual progress tracking.

Deployment & DevOps

  • DigitalOcean droplet running Ubuntu with firewall configured (UFW).
  • Gunicorn managed via systemd service, serving the Django application.
  • Nginx as reverse proxy using a Unix socket and custom site configuration.
  • Domain purchased via Squarespace with DNS pointing to the droplet.
  • App code versioned on GitHub, and deployed via git pull from the droplet.

Future Work — Machine Learning & Sensors

Custom Models on Real Training Data

As more users log workouts, I plan to build a dataset of anonymized exercise performance: sets, reps, weights, rest, volume, and progress over time. Using this data, I’ll train a simple model (e.g., via HuggingFace) that:

  • Suggests optimal progression (weight increases, rep ranges) for each user.
  • Flags potential overtraining or under-recovery based on volume and intensity trends.
  • Provides “next best workout” recommendations tailored to recent performance.

Sensor-Driven Training Insights

Long-term, I want to integrate motion sensors (e.g., Apple Watch, custom IMU hardware) to:

  • Detect eccentric vs concentric phases of each rep.
  • Measure time-under-tension and tempo consistency.
  • Identify dropsets, partial reps, and failure points automatically.
  • Score each set on “rep quality” instead of just counting reps.

This ties directly into my background in signals & systems and my interest in combining hardware, software, and AI into one feedback loop for athletes and lifters.
